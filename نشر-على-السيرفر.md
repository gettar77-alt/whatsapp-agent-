# دليل رفع مها على سيرفر Hetzner

دليل خطوة بخطوة لتشغيل وكيل واتساب "مها" على سيرفر Hetzner بحيث يعمل ٢٤ ساعة.
موجّه لصاحب السيرفر (شخص لديه خبرة بسيطة بالأوامر).

---

## نظرة عامة على النظام

- **مها** = تطبيق Flask (بايثون) يستقبل رسائل واتساب عبر **Webhook** من Meta Cloud API،
  يردّ عليها باستخدام Claude AI، ويحفظ المحادثات في ملف SQLite.
- المطلوب لتشغيلها دائماً:
  1. تشغيلها كخدمة دائمة (systemd + gunicorn)
  2. رابط **HTTPS عام** للـ webhook (Meta تشترط HTTPS بشهادة صحيحة)

---

## المتطلبات قبل البدء

| العنصر | ملاحظة |
|---|---|
| سيرفر Hetzner | يفضّل **Ubuntu 22.04 أو 24.04** |
| دومين (نطاق) | موجّه لعنوان IP السيرفر — **ضروري** لشهادة HTTPS. لو ما فيه دومين، شوف قسم "بدون دومين" بالأسفل |
| مفتاح Anthropic | `ANTHROPIC_API_KEY` من console.anthropic.com |
| بيانات WhatsApp | `WHATSAPP_TOKEN` + `WHATSAPP_PHONE_ID` + `WHATSAPP_VERIFY_TOKEN` من Meta |

> ⚠️ **مهم بخصوص توكن واتساب:** التوكن اللي يظهر في صفحة "Try it out" التجريبية في Meta **مؤقت وينتهي خلال ٢٤ ساعة**. للتشغيل الدائم لازم **توكن دائم (Permanent / System User Token)** — يُنشأ من Meta Business Settings ← System Users.

## الخطوات

### ١) الدخول للسيرفر
```bash
ssh root@SERVER_IP
```

### ٢) تثبيت الأدوات الأساسية
```bash
apt update && apt upgrade -y
apt install -y python3 python3-venv python3-pip git
```

### ٣) سحب الكود من GitHub
```bash
cd /opt
git clone https://github.com/gettar77-alt/whatsapp-agent-.git maha
cd maha
```

### ٤) إنشاء البيئة وتثبيت المكتبات
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install gunicorn
```

### ٥) إنشاء ملف الأسرار `.env`
```bash
cp .env.example .env
nano .env
```
املأ القيم الحقيقية:
```
ANTHROPIC_API_KEY=sk-ant-...
WHATSAPP_TOKEN=التوكن_الدائم_من_ميتا
WHATSAPP_PHONE_ID=رقم_الـ_Phone_ID
WHATSAPP_VERIFY_TOKEN=viewapartment2026
OWNER_PHONE=9665XXXXXXXX        # رقم المالك لإشعارات التحويل البشري
ADMIN_PASSWORD=كلمة_سر_قوية
```
احفظ بـ `Ctrl+O` ثم `Enter` ثم اخرج بـ `Ctrl+X`.

### ٦) تشغيل مها كخدمة دائمة (systemd)
أنشئ ملف الخدمة:
```bash
nano /etc/systemd/system/maha.service
```
الصق التالي:
```ini
[Unit]
Description=Maha WhatsApp Agent
After=network.target

[Service]
WorkingDirectory=/opt/maha
ExecStart=/opt/maha/venv/bin/gunicorn -w 1 --threads 8 -b 127.0.0.1:5000 app:app
Restart=always
User=root

[Install]
WantedBy=multi-user.target
```
ثم فعّلها:
```bash
systemctl daemon-reload
systemctl enable --now maha
systemctl status maha       # المفروض تشوف active (running)
```
> ملاحظة: نستخدم عاملاً واحداً (`-w 1`) مع threads لتفادي تعارض القفل في SQLite — كافٍ تماماً لحجم رسائل واتساب.

### ٧) HTTPS عبر Caddy (الأسهل — شهادة تلقائية)
ثبّت Caddy:
```bash
apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | tee /etc/apt/sources.list.d/caddy-stable.list
apt update && apt install -y caddy
```
عدّل إعداد Caddy:
```bash
nano /etc/caddy/Caddyfile
```
احذف المحتوى وضع (بدّل `your-domain.com` بدومينك الموجّه لـ IP السيرفر):
```
your-domain.com {
    reverse_proxy 127.0.0.1:5000
}
```
```bash
systemctl restart caddy
```
Caddy بيجيب شهادة HTTPS تلقائياً من Let's Encrypt. تأكد إن الدومين A record يشير لـ IP السيرفر، والمنفذين 80 و443 مفتوحين في جدار حماية Hetzner.

### ٨) ربط الـ Webhook في Meta
في لوحة Meta للتطبيق ← WhatsApp ← Configuration:
- **Callback URL:** `https://your-domain.com/webhook`
- **Verify token:** `viewapartment2026` (نفس اللي في `.env`)
- اضغط Verify and Save
- بعد التحقق، اضغط **Subscribe** على حقل **messages**

### ٩) الاختبار
- من Meta ← أضف رقم جوالك في "To" المسموح بها (للرقم التجريبي)
- أرسل رسالة للرقم → المفروض ترد مها

## بدون دومين؟ (Cloudflare Tunnel — مجاني)
لو ما فيه دومين، استخدم Cloudflare Tunnel للحصول على رابط HTTPS:
```bash
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
dpkg -i cloudflared.deb
cloudflared tunnel --url http://localhost:5000
```
بيطلع لك رابط مثل `https://xxxx.trycloudflare.com` — استخدمه كـ Callback URL في Meta.
> هذا الرابط للتجربة فقط ويتغير عند كل تشغيل. للإنتاج الدائم استخدم **Named Tunnel** عبر حساب Cloudflare (مجاني) مع دومينك.

## أوامر مفيدة للصيانة
```bash
systemctl status maha            # حالة مها
systemctl restart maha           # إعادة تشغيل مها
journalctl -u maha -f            # متابعة سجل مها لحظياً (لمعرفة الأخطاء)
cd /opt/maha && git pull && systemctl restart maha   # تحديث الكود لأحدث نسخة
```

## صفحات النظام بعد التشغيل
- `https://your-domain.com/`        → الموقع الإعلاني للشقق
- `https://your-domain.com/admin`   → لوحة الإدارة (أسعار + حجوزات + محادثات)
- `https://your-domain.com/demo`    → تجربة محادثة مها
- `https://your-domain.com/webhook` → نقطة استقبال واتساب (لا تُفتح يدوياً)
